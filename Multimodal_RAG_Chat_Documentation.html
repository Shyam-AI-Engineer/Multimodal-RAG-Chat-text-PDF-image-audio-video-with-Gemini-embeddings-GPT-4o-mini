1. Project Overview

This is a production-grade Multimodal Retrieval-Augmented Generation (RAG) system. It allows users to upload and ingest content in 5 modalities — plain text, PDFs, images, audio files, and video files. All content is embedded using Gemini Embedding 2 Preview via the Euri API and stored in a Pinecone serverless vector database. Users then ask natural-language questions through a ChatGPT-style UI; the system retrieves the most relevant content and generates a streaming answer using gpt-4o-mini (also via Euri).

Key design principle: The Euri API is the sole external AI provider. Both embedding and LLM generation share a single API key and base URL (https://api.euron.one/api/v1/euri), accessed through the standard OpenAI Python SDK. No Google SDK or OpenAI default endpoint is used.

5. Backend Deep-Dive

5.1 Entry Point — app/main.py

Creates the FastAPI application, wires up middleware and routers, and initialises the Pinecone vector store on startup via an asynccontextmanager lifespan event.

@asynccontextmanager
async def lifespan(app):
    vs = get_vectorstore()
    await vs.initialize()   # connects to Pinecone, creates index if missing
    yield

app = FastAPI(title="Multimodal RAG API", lifespan=lifespan)
app.add_middleware(CORSMiddleware, allow_origins=["http://localhost:3000"], ...)
app.include_router(ingest.router, prefix="/ingest")
app.include_router(query.router)

Utility endpoints exposed directly on app:

  • GET /health — returns {"status":"ok"}
  • GET /ingested — lists all ingested files with chunk counts
  • DELETE /ingested — removes vectors for a specific file from Pinecone

5.3 Euri Client — app/services/euri_client.py

Creates a single, cached OpenAI client instance pointed at the Euri base URL. Because it is decorated with @lru_cache(), the same object is reused across every request — no re-authentication overhead.

from openai import OpenAI

@lru_cache()
def get_euri_client() -> OpenAI:
    settings = get_settings()
    return OpenAI(
        api_key=settings.euri_api_key,
        base_url=settings.euri_base_url,  # overrides default openai.com endpoint
    )

5.4 Embedding Service — app/services/embedding.py

All five modalities are embedded using the same Euri endpoint. The core sync call is wrapped with tenacity retry logic (3 attempts, exponential back-off 2–30 s). Because FastAPI is async but the OpenAI SDK's embeddings.create() is synchronous, all embedding calls are offloaded to the default thread-pool executor via loop.run_in_executor(None, ...).

Non-text modalities (images, audio, video) are base64-encoded and passed as data URIs in the format data:<mime>;base64,<data> — the same text input field that accepts strings. The Gemini model natively understands these data URIs.

# Example: how images become embeddings
inputs = [f"data:{mime_type};base64,{base64.b64encode(img).decode()}"
          for img in image_bytes_list]
response = client.embeddings.create(model="gemini-embedding-2-preview",
                                    input=inputs, dimensions=768)

5.5 Vector Store — app/services/vectorstore.py

Wraps the Pinecone Python client. Uses one namespace per source type to allow optional filtering during queries.

NamespaceSource Type
ns_textPlain text
ns_pdfPDF documents
ns_imageImages (PNG/JPEG)
ns_audioAudio files (MP3/WAV)
ns_videoVideo files (MP4/MOV)

Key methods

  • initialize() — connects to Pinecone; creates the index (dim=768, metric=cosine, serverless on AWS us-east-1) if it does not already exist.
  • upsert_vectors() — builds vector records with rich metadata (source_type, source_file, chunk_index, content_preview, timestamp) and upserts in batches of 100.
  • query() — queries one namespace (if source_type filter given) or all 5 namespaces, then merges and re-ranks by score to return top-k SourceReference objects.
  • delete_file() — fetches up to 1000 matching vector IDs by file name and deletes them.
  • list_ingested_files() — for each namespace, probes stats and samples vectors to reconstruct a list of ingested files.
# Vector ID format: <type>_<filename>_<chunk_idx>_<8-hex-random>
vector_id = f"{source_type}_{source_file}_{chunk_idx}_{uuid.uuid4().hex[:8]}"

5.6 LLM Service — app/services/llm.py

Two generation modes: synchronous JSON response and async SSE streaming. Both use the same system prompt that strictly instructs the model to answer only from retrieved context.

System Prompt (verbatim)

You are a helpful AI assistant that answers questions based ONLY on the provided context.
Rules:
1. Answer using ONLY the information found in the context.
2. If context is insufficient, say so — do NOT make up information.
3. Be concise, accurate, and helpful.
4. You may mention the source file name.
5. Do not hallucinate facts not in the context.

User Message Construction

Retrieved chunks are formatted and prepended to the user's question:

Context:
[Source 1] File: report.pdf (Type: pdf, Chunk: 0)
<first 200 chars of chunk 0>

[Source 2] File: notes.txt (Type: text, Chunk: 3)
<first 200 chars of chunk 3>

Question: What is the conclusion?

Streaming SSE Protocol

The stream yields three event types:

  • data: {"content": "<token>"} — one per LLM output token
  • data: {"sources": [...]} — full source references, sent after stream ends
  • data: [DONE] — terminal signal

Video Description

describe_video_frames(frames, filename) is a special vision call that sends up to 6 JPEG frames extracted from a video to the LLM with a multi-part prompt asking it to describe topics, key concepts, visible text/diagrams, and the overall narrative. The resulting text description is then chunked and embedded as text vectors.

5.7 RAG Pipeline — app/services/rag_pipeline.py

Thin orchestration layer. The two public functions are called by the routers:

async def run_query(question, source_type, top_k) -> QueryResponse:
    query_vector = await embed_single_text(question)
    sources      = await vs.query(query_vector, top_k, source_type)
    answer       = await generate_response(question, sources)
    return QueryResponse(answer=answer, sources=sources, question=question)

async def run_query_stream(question, source_type, top_k) -> AsyncGenerator[str]:
    query_vector = await embed_single_text(question)
    sources      = await vs.query(query_vector, top_k, source_type)
    async for chunk in stream_response(question, sources):
        yield chunk

5.8 Processors

Text Processor (text_processor.py)

Uses RecursiveCharacterTextSplitter from langchain-text-splitters. Converts token limits to character counts (1 token ≈ 4 characters) and splits on ["\n\n", "\n", ". ", " ", ""] in priority order. Returns TextChunk dataclass objects with text, chunk_index, and content_preview.

chunk_size_chars  = 1024 * 4  # = 4096 chars
chunk_overlap_chars = 256 * 4 # = 1024 chars

PDF Processor (pdf_processor.py)

Uses PyPDF2.PdfReader to extract text page-by-page. Groups pages into batches of up to 6 (Gemini's PDF limit). If a page has no extractable text (image-only scanned PDF), it inserts a placeholder string rather than silently dropping it. Returns PDFPageBatch dataclass objects.

Image Processor (image_processor.py)

Validates file extensions (.png, .jpg, .jpeg). Groups uploaded images into batches of up to 6. Returns ImageBatch objects containing ImageItem entries with image_bytes, mime_type, and filename.

Audio Processor (audio_processor.py)

Validates .mp3/.wav files and wraps them in an AudioItem dataclass. No transcription is performed — audio is passed directly to the embedding model as a base64 data URI.

Video Processor (video_processor.py)

Two code paths:

  1. With ffmpeg available (_try_ffmpeg_segment): Uses ffprobe to measure exact duration, then ffmpeg to cut accurate 120-second segments.
  2. Without ffmpeg (fallback): Estimates duration from file size at 1 Mbps and splits by byte ranges (not frame-accurate).

Frame extraction (extract_frames_as_jpeg) uses the bundled binary from imageio-ffmpeg. It probes duration from ffmpeg stderr, then extracts up to 6 evenly-spaced JPEG frames via individual ffmpeg -ss ... -vframes 1 subprocess calls.

5.9 Routers

Ingestion Router (routers/ingest.py)

Five POST endpoints. Each follows the same pattern:

1
Read uploaded bytes
2
Validate (extension, size, non-empty)
3
Run the appropriate processor → chunks/batches/segments
4
Call the appropriate embedding function
5
Upsert to Pinecone (initialise if not yet done)
6
Return IngestResponse with chunk count

Special: Video ingestion adds an extra step between 3 and 4:

# Extract frames → vision LLM describes the video → embed the text description
for segment in segments:
    frames      = extract_frames_as_jpeg(segment.video_bytes, filename)
    description = await describe_video_frames(frames, filename)
    chunks      = chunk_text(description, source_name=filename)
    vectors     = await embed_texts([c.text for c in chunks])

Query Router (routers/query.py)

  • POST /query — delegates to run_query(), returns QueryResponse JSON.
  • POST /query/stream — delegates to run_query_stream(), wraps the async generator in a StreamingResponse with media_type="text/event-stream" and cache-control headers for SSE compatibility.

6. Frontend Deep-Dive

6.1 TypeScript Types — src/lib/types.ts

Defines interfaces mirroring every backend Pydantic model plus UI-specific state types. Key types:

  • ChatMessage — id, role, content, sources?, isStreaming?, error?, timestamp
  • UploadItem — per-file upload state: status (idle/uploading/processing/success/error), progress, message
  • SSEContentEvent, SSESourcesEvent, SSEErrorEvent — discriminated union for parsing SSE stream

6.2 useChat Hook — src/hooks/useChat.ts

Manages the entire chat session state using useState. Uses an AbortController ref to support mid-stream cancellation.

State managed

  • messages: ChatMessage[] — full conversation history
  • isLoading: boolean — true while streaming
  • error: string | null

Flow when user sends a message

1
Append userMessage and a placeholder assistantMessage (isStreaming=true) to state
2
Call queryStream() from api.ts with the AbortController signal
3
onToken callback: appends each token to assistantMessage.content via immutable state update
4
onSources callback: sets assistantMessage.sources
5
onDone callback: sets isStreaming=false, clears loading state
6
onError callback: sets error message in message and global error state

7. End-to-End Data Flow

7.1 Ingestion Flow

1
User drags/drops a file (or pastes text) on the Ingest page
2
Frontend detects modality from extension → calls the correct POST /ingest/<type>
3
Backend validates the file, runs the appropriate processor to split into chunks/batches
4
Chunks are embedded via embedding.py → Euri API → Gemini Embedding 2 Preview → 768-dim vectors
5
Vectors upserted to Pinecone in batches of 100 with full metadata
6
JSON response returned to frontend; IngestionStatus component shows success

7.2 Query / Chat Flow

1
User types question in ChatInput and presses Enter
2
useChat.sendMessage() adds placeholder messages and calls queryStream()
3
Frontend POSTs to /query/stream with question, optional source_type filter, top_k=5
4
Backend embeds the question → 768-dim query vector
5
Pinecone queried across all (or filtered) namespaces; top-5 chunks returned
6
Prompt built: system instruction + formatted context + question
7
LLM called with stream=True → tokens yielded as SSE data: {"content": "..."} events
8
After all tokens, SSE data: {"sources": [...]} then data: [DONE]
9
Frontend useChat accumulates tokens into the assistant message; sources appear below
